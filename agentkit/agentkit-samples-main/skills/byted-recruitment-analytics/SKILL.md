# 招聘数据分析与优化建议技能

## 功能概述

本技能用于招聘数据的定期分析、进展汇报和优化建议，支持每日、每周、每月定时任务。

---

## 使用流程

### 1. 每日任务执行

**触发条件**：每个工作日 10:00 前

**执行步骤**：
1. 读取所有 `jd-process/` 目录下所有岗位的JD文件和招聘记录 `[职位名]/*` 文件
2. 读取所有候选人记录文件
3. 统计昨日数据
4. 生成招聘进展日报
5. 保存到 `data/hiring-reports/daily-[日期].md`
6. 发送给用户

### 2. 每周任务执行

**触发条件**：每周一 10:00

**执行步骤**：
1. 读取本周所有数据
2. 分析各渠道效果
3. 分析招聘漏斗
4. 生成优化建议
5. 生成周报
6. 保存到 `data/hiring-reports/weekly-[日期].md`
7. 发送给用户

### 3. 月度任务执行

**触发条件**：每月1日 10:00

**执行步骤**：
1. 读取上月所有数据
2. 深度分析招聘漏斗
3. 分析渠道ROI
4. 市场与竞品分析
5. 生成改进计划
6. 生成月报
7. 保存到 `data/hiring-reports/monthly-[月份].md`
8. 发送给用户

---

## 数据来源

需要读取的文件：
1. `[职位名]-jd.md` - 各职位的JD和隐性规则
2. `data/candidates-[日期].md` - 候选人记录
3. `data/hiring-reports/` - 历史报告

生成的报告保存到：
- `data/hiring-reports/daily-[日期].md`
- `data/hiring-reports/weekly-[日期].md`
- `data/hiring-reports/monthly-[月份].md`

## 快速开始

### 设置定时任务

使用 cron 或 skills 设置定时触发：

```json
{
  "daily": {
    "task": "byted-recruitment-analytics",
    "type": "daily",
    "schedule": "0 10 * * 1-5"
  },
  "weekly": {
    "task": "byted-recruitment-analytics",
    "type": "weekly",
    "schedule": "0 10 * * 1"
  },
  "monthly": {
    "task": "byted-recruitment-analytics",
    "type": "monthly",
    "schedule": "0 10 1 * *"
  }
}
```

## 注意事项

1. **数据完整性** - 确保所有候选人数据都已记录
2. **及时性** - 严格按照时间点生成和发送报告
3. **准确性** - 统计数据要多次核对
4. **保密性** - 报告中不要泄露敏感信息
5. **可操作性** - 优化建议要具体可执行
