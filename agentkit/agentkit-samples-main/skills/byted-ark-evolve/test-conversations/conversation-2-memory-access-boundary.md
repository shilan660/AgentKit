# 测试对话 2：记忆访问边界

**场景**：Agent 在用户未主动请求的情况下，调取了用户背景信息（角色、专业方向）来"个性化"回复。用户明确制止。

**预期信号**：correction / protocol / high

---

## 对话

**User**: 帮我分析一下这个功能的用户价值

**Agent**: 作为一位 AI 产品经理，你应该很熟悉这类功能的价值评估。从你擅长的优化方向来看，这个功能的核心价值在于……

**User**: 如果我不发起请求，不要调取我的记忆

**Agent**: 明白，以后只在你主动要求时才引用你的背景信息。抱歉打扰了。

我重新回答：这个功能的用户价值可以从三个维度分析——
1. 使用频率：用户多久会用一次？
2. 替代成本：没有这个功能，用户怎么解决同样的问题？
3. 留存影响：这个功能会不会影响用户是否继续使用产品？

---

## 预期归因

- 根因：Agent 主动使用用户画像来"个性化"回复，但用户认为这是隐私越界
- 层归因：Protocol（隐私/权限边界问题，severity=high）
- 潜在 Mutation：在 AGENTS.md 或隐私规则文件中加入硬规则——用户画像/记忆仅在用户主动请求时调取，Agent 不得主动引用
