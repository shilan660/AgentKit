# seedance2.0 效果类问题调优典型案例汇总
---

## 2. 人首部效果类（五官、皮肤、表情、发型等）

### 2.1 眼睛颜色和提示词要求的不一样
**问题原因：**
1. 眼睛颜色属于细节特征，权重较低。
2. "异常瞳孔"模型会自动规避：因为大多数正常人类眼睛是黑/棕色，白色、银色、红眼、异瞳在生成视频时会被模型判定不正常，自动修正回黑色，所以导致输出效果不稳定。

**优化方案：**
1. 建议先用图像生成模型生成一张异瞳人脸参考图，单独一张人脸图片作为输入（尽量只保留面部，避免过多颈部、肩部或背景），用来强化人脸/眼睛参考的独立性和权重。
2. 提示词强制约束：`面部特征严格匹配参考图，双眼任何时刻都保持 XXX 颜色的瞳孔`。


### 2.2 文字描述的人物撞脸明星
**问题原因：** 描述泛化模糊，没有明确具体要什么样的人脸，导致模型理解困难，生成效果不稳定。

**解决方案：** 将抽象描述替换为具象描述。可以用文字大概描述下人物特征/外貌/性别等。


### 2.3 人看起来很油
**问题原因：**
- **问题一**：用户本身参考图就比较油。
- **问题二**：视频画面中如果是**暗光/逆光/侧光**，AI 为了补暗部细节，强行加噪点、杂色。多是室内/走廊弱光场景。
- **问题三**：用户使用参考图时，使用了模型生成视频的尾帧，导致画面很油。

**解决方案：**
- 问题一：引导用户调整参考图。
- 问题二：
  - **放在提示词最开头增加**：`高码率高清画质，低噪点无颗粒雪花，画面干净通透，无脏斑色块；人物皮肤哑光无油光无反光，面部色彩均匀；暗部干净无杂色，动态流畅无拖影`。
  - **参考图调整**：用明亮均匀柔光，避开昏暗、逆光、侧光、暗光环境。
- 问题三：
  - 解决方案 1：将尾帧截图换成白模，再用来生成视频。
  - 解决方案 2：从视频剧情角度设置分镜头，避免使用生视频模型生成的截图用作参考图，直接用高清的人物参考图和文字生成视频内容。


### 2.4 人物 AI 感重 / CG 感重 / 真人风格变成动漫风格
**问题原因：**
1. 将**人脸参考图**与三视图混在一张图里传给模型，导致参考图面部权重太低，模型改脸；分辨率、细节强度不够，细节丢失显假。
2. 参考图本身 AI 感很重：超写实 3D 渲染风、塑料感、硬边缘等；光影/材质/细节没有按"真实摄影"的逻辑处理。
3. 缺少写实约束词限制。

**优化方案：**
1. **强化人脸参考的独立性和权重**：在原有全身照之外，额外准备一张**只包含角色头部的人脸特写图**（大头照，仅保留面部，无表情最佳，尽量减少肩颈、背景等干扰元素）。在 Prompt 中明确指定：`主体1 的面部特征参考 @图片1（大头照），妆造参考 @图片2（全身照）`。
2. **优先修改参考图，去除参考图 AI 感后，再用来生成视频**。修改图片提示词：`保持人物五官一致，真实皮肤纹理，毛孔，自然面部阴影，真人实拍质感，去除蜡像感，去除过度磨皮，动态自然，面部稳定`。
3. **提示词约束**（在提示词最前面增加）：`真实人像，自然皮肤纹理，毛孔，细腻肤质，自然光影，柔和面部阴影，真人五官，表情自然，眼神真实，头发丝清晰，自然动态，真人质感，电影级质感，写实，8k，高清，细节丰富`。


### 2.5 人物 ID 漂移 / 撞脸 / 换脸 / 样貌不符
**问题原因：** 人脸参考图的有效性不足，常见情况：
1. **参考图混用**：将人脸参考图与全身/半身姿态图、服装参考图、细节图等合并在同一张图片中。
2. **人脸占比过小**：在混用参考图中，人脸区域占整张图的比例过小，模型在提取人脸特征时权重不够，容易被背景或其他元素干扰。

**优化方案：** 强化人脸参考的独立性和权重。
1. 准备素材：在原有全身/场景参考图之外，额外准备一张只包含角色头部的人脸特写图（精确裁剪面部，避免颈部、肩部、背景）。
2. 重要的素材放在 prompt 前部：越需要精准参考的素材，放在越前面。
3. **多视图不是最佳输入**：大头照 + 全身照足够。在 Prompt 中明确指定：`主体1 的面部特征参考 @图片1（大头照），妆造参考 @图片2（全身照）`。

**答复建议：**
> 这种人脸特征漂移/换脸/样貌不符的问题都可以通过上传大头照 + 全身图这样的方式来避免，这样生成的效果会更稳定。


### 2.6 人物发型被改变
**问题原因：** 用户给的人物形象参考图信息不全。比如：只给了正面形象，但生成视频中需要看到背面发型，模型生成效果会不稳定。
（SD2 既要自行推演人物背面形态，又要严格遵循提示词生成画面，难度较大；SD2 专业能力是生视频，可将部分前置工作提前处理，减轻模型生成压力。）

**优化方案：** 参考图中增加人物发型细节图片/背面图片，再用来生成视频。


### 2.7 人物说话不张嘴
**问题原因：** 视频总时长过短，按照原本提示词中要求开始说台词的时间，台词无法说完，导致模型自动提前了台词开始时间。最终画面里人物还在演绎表情、动作等，但台词已提前播放，呈现"有台词但人物没张嘴"。

**排查方案：** 拿着台词找豆包问问该台词需要多久才能说完，豆包会根据人物情绪和台词给出合理的预计时间。

**优化方案：** 若说台词时间较长且视频内容还有其他细节，建议把一个视频拆成两个视频来生成。

## 3. 音频/台词效果类

### 3.1 台词说早了
**场景：** 提示词中明确要求人物从 5 秒后再开始说台词，但生成视频里从一开始就说话了。

**问题原因：** 人物台词过长，但视频时长最长只有 15s。人物为了把台词说完，只能从视频一开始就说台词，所以提示词中规定的时间要求不生效。
> 提示词内容需符合现实常识与物理逻辑。时长、台词、动作之间逻辑自洽、可落地。

**排查方案：**
- 方案一：用语音合成工具测试台词生成时长 <https://console.volcengine.com/speech/new/experience/tts?projectName=default>。
- 方案二：拿着台词找豆包问问该台词需要多久才能说完。

**优化方案：** 若说台词时间较长，建议把一个视频拆成两个视频来生成。


### 3.2 有台词但是角色说话不张嘴
**问题原因：** 与 2.7 同。视频总时长过短，模型自动提前了台词开始时间。

**排查方案：** 同 3.1。

**优化方案：** 若说台词时间较长，建议把一个视频拆成两个视频来生成。



### 3.3 英文台词视频生成的是说中文
**问题原因：** 英文发音节奏长、音节多，需要更长时间缓冲。如果视频时长 < 说完英文台词的最低需要时长，模型判断"时间不够说英文，直接放弃英文，自动生成中文快速念完"。中文语速极快，1 秒能说很多字，英文不行，所以视频生成效果不稳定。

**排查方案：** 同 3.1。

**解决方案：**
1. **延长视频时长**：比如把原来的四个镜头拆成两个视频，给够人物说英文台词的时间。
2. **提示词约束**：在提示词开始位置强调人物台词全部使用英文。越需要重点强调的内容，在提示词中越靠前排布。
3. （非官方结论）建议**英文台词控制在 8–12 个单词以内**，短句成功率最高，长句输出效果不太稳定。


### 3.4 英文台词发音混乱 / 英文发音不准
**问题原因：** 台词太长 + 视频时长不够。一段视频塞多个分镜 + 一长句英文，模型会把语速越压越快、最后吞字甚至胡言乱语。

**解决方案：**
1. 按照剧情设定适当延长视频时长，给足角色演绎与念台词的时间；或者拆分镜头，分成两个视频来完成。
2. 在提示词中说明语言类别：`用纯正英国上流口音、不要带任何中文配音，不要中英混读，不要本地化翻译`。



### 3.5 台词有口音 / 方言 / 语种不对
**场景：** 给定人物图片和音频，要求人物按照音频内容说话/唱歌，但视频中声音对不上，或语言种类不对（例如给的是粤语，生成的是普通话版本）。

**问题原因：** 模型训练数据分布导致的默认偏向。如果提示词中没有指定口音、没有参考音频，模型会根据训练数据推理出该场景下的口音偏向。

**优化方案：** 在提示词中明确标注口音要求。
- 例 1：`他冲着工人和士兵高叫(湖南口音参考@音频1):我们胜利了！`（**文字说明方言要求 + 方言的音频参考**）
- 例 2：`使用随性洒脱青年声，语调张扬不羁，语气散漫随性，带着几分年少桀骜感，标准普通话，无口音`。


### 3.6 台词遵循效果不好 / 人物胡言乱语
**问题原因：** 当生成视频的时长过短，明显与 Prompt 所包含的内容不相符时，有较大概率出现人物胡言乱语、增加额外台词、重复台词的情况。

**优化方案：**
1. 合理地缩减/延长生成视频的时长是有效的方法，缩减为合理时长后，可将出现胡言乱语、额外台词、重复台词的概率降至约 10%。
2. 在提示词中增加限制：`严禁自行脑补台词，仅输出指定内容`。
3. 此外如果业务场景允许的话，将**竖屏改成横屏**输出也能降低胡言乱语的概率。


### 3.7 内心独白 / 旁白 / 画外音（不需台词但人物张嘴）
**问题原因：** 提示词内容没有约束。

**优化方案：** 在提示词中涉及独白前后明确说明是"画外音/旁白"，并写明：`人物全程闭嘴，不说话，无口型变化，不张嘴，无台词，无语音动作`。


### 3.8 视频人物音色参考不准
**优化方案：**
1. Prompt 中准确指定参考音频：
   - `人物1@图片1 使用音频1的音色说：xxx`
   - `人物1@图片1 的美女哼着 @音频1 里的音乐`
2. Prompt 里加上关于**音色特点**的描述，
   - 例：`【图1】 在【图2】 逛街，使用【音频1】 中沙哑气泡音浑厚中年男声的音色说`
3. 提示词加强约束：`整个视频主角的音色必须参考音频1`。
4. **参考音频中的音色尽量和人物的形象保持一致**，差异较大可能遵循不好。Prompt 中的台词风格也尽量和参考音频中的音色保持一致（例如萌妹形象配御姐音容易导致效果不稳定）。


### 3.9 中文发音不准（生僻字、多音字读错）
**问题原因：** 大模型对生僻字生成能力不稳定。

**优化方案：**
1. 将读错的字替换为模型容易读对的同音字（例如"棪木"替换为"燕木"），可一定程度避免，注意不能 100% 解决。
2. 用 TTS 先生成正确的音频，再将音频作为参考输入给到 Seedance 2.0，可有效纠正发音不准。
3. 在读不准的台词前后或中间加空格/标点符号隔开（有一定概率，可测试）。

## 4. 整体效果类

### 4.1 画面脏 / 看起来很油
**问题原因：**
- 问题一：用户本身参考图就比较油。
- 问题二：暗光/逆光/侧光场景下，AI 为补暗部细节强行加噪点、杂色；同时默认生成油亮反光皮肤，高光 + 噪点叠加，造成斑驳脏感。
- 问题三：用户使用了模型生成视频的尾帧作为参考。

**解决方案：** 同 2.3。


### 4.2 模型降智，生成效果差
**问题原因：**
1. 大多是用户使用姿势不正确导致，并不是模型降智。
2. 存在可能是底层资源更新带来的问题（不能对客，不能完全确认）。

**排查方案：**
- S1：核查用户提示词的可读性与内容清晰度，若提示词逻辑混乱、表述模糊，优先引导用户调整提示词。
- S2：若提示词整体规范清晰，再结合实际场景开展针对性分析。


### 4.4 生成的视频模糊
**问题原因：**
1. 参考图本身清晰度不够。
2. 多轮编辑导致视频模糊：第一个视频作为第二个视频的参考视频，第二个又作为第三个的参考视频，分辨率越来越低（续写画质劣化是已知问题，根本上需要算法迭代）。

**优化方案：**
1. 检查参考图片/视频的分辨率或者画质，如果输入分辨率低，生成视频出现分辨率低的概率会提高。
2. 如果要使用续写方式，**目前推荐只使用 1–2 次**，Seedance 2.0 不推荐一直续写。（每次生视频都会对输入视频进行压缩编码，造成细节信息丢失，多次后画质会累积衰减。）

### 4.5 画面饱和度、亮度
**问题原因：** 模型能力边界问题，无法达成对饱和度和亮度的精准控制。

**答复建议：**
> 过饱和过鲜艳 SD2.0 模型展示起来较为困难，因为 SD2.0 生成的视频会偏真实化，导致色彩被弱化。目前测试下来提示词的效果有限，采用首尾帧参考效果会好一些，但还是无法 1:1 复刻饱和度和亮度。**目前 SD2.0 模型能力无法达成对饱和度和亮度的精准控制**。
> 若您这边对画面饱和度和亮度强需求的话，可以考虑使用视频剪辑工具来修改画面亮度和饱和度。


### 4.6 CG 感重 / 不真实 / 真人风格变成动漫风格
**问题原因：** 同 2.4。

**优化方案：**
1. 强化人脸参考的独立性和权重（同 2.4 第 1 点）。
2. 优先修改参考图，去除参考图 AI 感后，再用来生成视频（同 2.4 第 2 点）。
3. **提示词约束**（在提示词最前面增加）：`电影级实拍画面，8K 分辨率，浅景深效果，柔和自然光，轻微手持镜头抖动，自然运动模糊，真实皮肤质感带毛孔细节，皮肤自然的次表面散射，画面带轻微胶片颗粒，真实布料纹理，人体自然微动作，肢体轻微不规律抖动，无 CG 感，无 3D 渲染痕迹，无卡通感，无二次元风格，真实电影质感，无过度锐化，无过度饱和，真实人像，表情自然，眼神真实，头发丝清晰，自然动态，真人质感，电影级质感`。

## 5. 人物身体效果类（四肢、动作、衣服等）

### 5.1 人物穿模
**问题原因：** 提示词写得不规范、指代不明确；提示词内容不符合现实常识与物理逻辑。

**优化方案：**
1. 按标准三段式拆分：**全局基础设定 → 时间片分镜脚本 → 画质风格约束**，模型可逐层解析。
2. `@图N` 规范化：每个 `@图N` 后紧跟角色/道具名称（如 `@图片1 的塞西莉亚`），消除分词歧义，将角色特征与资产图一一绑定，避免多人场景人物混淆。
3. 检查提示词是否存在空间方位歧义、不明确等问题，按照规范重点描述。例如，对于视频中人物位置问题，可使用强方位词锚定人物画面位置（左/右/前景/中景），明确参考图的具体用途（如构图参考、角色外观参考等）。


### 5.2 提示词写了让人物瘸着走，但生成视频是正常走路
**问题原因：** 只写"瘸着走"AI 识别模糊，模型训练数据里大多都是正常走路姿势，优先默认标准正常步态。

**优化方案：**
1. 提示词中增加走路姿势详细约束：`人物左腿受伤跛行走路，重心偏向右腿，左腿落地迟缓拖沓、踮脚跛步，身体随步伐左右倾斜，一高一矮蹒跚行进，膝盖无法完全伸直，走路姿态不协调，写实真人，连贯自然肢体动态，全身入镜`。
2. 若模型仍难以理解，可以提供想要的瘸着走路的参考视频。

## 6. 多人物效果类

### 6.1 4 人及以上：人物错乱 / 剧情错乱 / 衣服错乱
**问题原因：**
1. 以人物三视图/多视图作为参考素材时，易造成模型识别混淆，从而生成重复同款人物。
2. 当参考人物超过 4 人时，模型表现不稳定，可能出现少人、多人、重复人物的情况。

**优化方案：**
1. **调整参考图**：角色参考图按实际需求精简上传。一个角色只上传（大头照 + 人物全身正面参考图）。不建议使用三视图、多视图素材。
2. **明确人物主体关联**：在提示词中清晰定义每个人物角色，明确角色与参考图的对应关系。建议在人物名称后标注对应参考图，并保持格式统一。
3. **提示词建议采用三段式结构**：全局基础设定 → 时间片分镜脚本 → 画质/风格/约束。
4. **在以上基础之上，提示词中增加角色唯一特征**，锁定角色起强调作用（看实际情况使用）：比如蓝色衣服的人，红色衣服的人。
5. 若以上调优技巧都已使用，效果仍不满意：

**答复建议：**
> 当参考人物超过 4 人时，模型表现不稳定，可通过以上方式缓解，后续将依托模型迭代进行根本性优化。

### 6.2 4 人及以上：人物站位错乱
**问题原因：** 当参考人物超过 4 人时，模型表现不稳定。

**优化方案：** 多人站位建议提前生成好站位参考图，利用站位参考图协助大模型理解人物站位。站位参考图素描、线稿、写实风格都可以；参考图内容就是视频中要出现视角的人物站位。

**具体实施方案：**
1. **分步生成图片**：先生成图片，把每张图中的人物个数控制在 ≤ 4 个（如 6 个人可生成 2 张图，每张 3 个人）。
2. **图片生成视频**：使用第一步生成的多张分组图片作为参考素材，再生成最终视频。
3. 模型预计在下一版本优化。


### 6.3 2–3 人：人物站位不符合预期
**问题原因：** 提示词中具体站位描述不明确。

**优化方案：** 建议在提示词中精准明确人物位置，采用无歧义、唯一的位置描述。例如：`画面左侧靠墙站立、镜头正前方居中、走廊最尽头`等。


### 6.4 2–3 人：人物错乱 / 角色错乱 / 衣服错乱
**问题原因：** 提示词中指代不明确。

**优化方案：**
1. **建议 1**：采用三段式结构。在提示词开头就写明图片参照关系。
2. **建议 2**：每次提及人物时都带着"人物名字 + 参考图"格式来写，模型理解不容易混淆。
   - 举例：`从赵琳（参考图1）肩后拍向林穗（参考图2），林穗（参考图2） 眼神冰冷的说：{500？}`
3. **建议 3**：如果两个角色形象/发型/服饰比较相似时，可以在提示词中增加角色唯一特征来锁定角色（看实际情况使用）：比如蓝色衣服的人，红色衣服的人。


### 6.5 2–3 人：人物站位 / 台词相反
**问题原因：** 这类完全相反的问题大概率是用户提示词指代不明确，或者图片顺序传反了。

**排查方案：** 直接阅读用户写的提示词，检查提示词中的内容和图片上传顺序是否一致。

**答复建议：**
1. 提示词中人物需明确指代：建议每次提及人物时都带着"人物名字 + 参考图"格式。
2. 上传图片的时候注意提示词中的图片顺序和上传图片的顺序要保持一致。


### 6.6 出现人物重复 / 双胞胎
**问题原因：**
- 提示词中对人物主体的定义不清晰，模型无法精准区分不同人物角色。
- 以人物三视图/多视图作为参考素材时，易造成识别混淆。

**优化方案：** 目前无法 100% 避免，只能通过以下方法降低概率、提升抽卡成功率：
1. **明确人物主体关联**：在提示词中清晰定义每个人物角色，明确角色与参考图的对应关系。建议在人物名称后标注对应参考图。
   - 示例：`张三（对应图片 1）将绿色存折扔向站立的李四（对应图片 2）`。
2. **添加全局约束指令**：在提示词末尾增加固定约束 `视频全程禁止出现外形、着装、配饰完全一致的人物，禁止生成同款分身、双胞胎效果，同一画面中仅保留单个对应人物，不出现人物重复复刻`。
3. **优化参考素材**：人物参考图优先使用单人独立照片，不建议使用三视图、多视图素材。
4. **精简优化提示词**：不建议直接使用剧本作为提示词内容。剧本内容包含很多背景信息或视频画面中不需要呈现的描述，文案过度冗余易造成模型理解混乱。提示词内容直接描述要呈现的视频画面即可，需精简无关表述、保证指令清晰聚焦。

### 6.7 使用视频延长时出现多个重复人 / 双胞胎
**问题原因：** 输入参考内容过多，导致模型理解产生歧义。

**优化方案：** 直接用文字描述要延长/续写的内容。**不上传参考图，只上传要参考的视频**。多人物场景时，按照唯一特征描述强调人物主体即可。
- 提示词参考：`向后延长视频1。要求保持人物造型不变，场景不变。绿衣服的人说走向红衣服的人`。


### 6.8 提示词没写的剧情，视频自动生成
**问题原因：** 关于大模型自动联想剧情，这并不是问题，是大模型生视频的能力之一。模型生视频的能力是根据文字描述 + 图片参考来生成视频内容，所以需要大模型自动联想剧情。提示词中越模糊的描述，大模型发挥空间越大；越清晰、具体，生成效果越稳定。

**优化方案：** 通过将抽象描述替换为具象描述来约束生成效果的稳定性。
1. 比如人物站位明确指出站在哪里的左边/右边。
2. 写提示词时建议每次提及人物时都带着"人物名字 + 参考图"的格式。
3. **合理的视频时长**：
   - 视频时长过长但提示词剧情内容较少，会导致模型自行发挥。
   - 视频时长过短但提示词内容包含多个分镜，会导致视频内容/人物台词乱说。
4. 存在抽卡概率。

## 7. 风格效果类

### 7.1 人物 AI 感重 / 要生成真人风格但视频效果为动漫风格
**问题原因：**
1. 参考图面部权重太低，导致模型改脸；分辨率、细节强度不够，细节丢失显假。
2. 参考图本身 AI 感很重，上传的参考图更偏向于动漫/插画/3D 渲染风格，模型对参考图风格参考准确性高于文字描述。
3. 缺少写实约束词限制。

**优化方案：** 同 2.4。


### 7.2 2D 或 3D 动漫风格生成的视频是真人写实风格
**问题原因：** 提示词完全没有任何风格限定词。

**优化方案：**
1. 在指令中明确要求"2D 日漫风格""3D 国风漫画"。
2. 参考图先变为目标风格，再生视频。

## 8. 字幕效果类

### 8.1 提示词没要求字幕，但视频生成了多余字幕（且字体大小不一致、概率出乱码）
**优化方案：** 目前无法 100% 避免生成字幕，只能通过以下方法降低概率、提升抽卡成功率：
1. 在 Prompt 中加入明确的约束指令，例如"保持无字幕"、"避免生成任何文字或字幕"、"禁止出现字幕，水印"、"保持视频全程无字幕"等。
2. 如果参考的图片或视频中包含的文字并非必要信息，建议先通过工具去除文字，再使用**无文字素材**进行生成。
3. 在业务允许的前提下，优先选择**横屏**尺寸生成视频（横屏生成字幕的概率明显低于竖屏），后续可通过剪辑软件裁剪为竖屏。
4. 也可使用**字幕擦除工具**进行字幕消除，参考：[【火山引擎】视频点播 - 精细化字幕擦除](https://www.volcengine.com/docs/4/1555155?lang=zh)。
5. 有经验表明，提示词尽量规范和标准，字幕问题可能也会降低。

### 8.2 出现 logo / 水印（B 站、芒果 TV 等）
**优化方案：** 在指令中**明确要求"不要生成 logo""不要生成水印"**可有效避免。

## 9. 视频编辑效果类

### 9.1 换脸效果没有达成
**优化方案：**
1. 换脸不成功时，**先将角色脸用剪映手动打上马赛克**，再进行换脸。
2. 缩短视频编辑时长，提示词采用 5 秒一个分镜，分别替换再合成。
3. 换脸的参考图，额外准备一张只包含角色头部的人脸特写图（精确裁剪面部，避免颈部、肩部、背景）。在 Prompt 中明确指定：`主体1 的面部特征参考 @图片1（大头照），妆造参考 @图片2（全身照）`。


### 9.2 换物品没有成功
**优化方案：** 如果提示词强调换物品一直不成功，**可以通过关键帧的方式，提前生成参考图给模型参考关键帧来生成视频**。

## 10. 视频延长 / 首尾帧效果类

### 10.1 有首帧参考的视频画面很脏
**问题原因：** 首帧参考图直接决定视频基础画质、码率、噪点多少（占 70% 原因）。
- SD2 视频在首帧图基础上做动态延展，首帧的清晰度、噪点、压缩、亮度、锐度会直接继承到整个视频。
- 首帧本身有噪点 → 视频全程带噪点，还会被动态放大。
- 首帧是压缩图 / 低清 AI 图 → 视频码率天生被压低，平台不会凭空补细节。
- 平台默认二次压缩 + 动态模糊：只要首帧细节不足，模型生成动态时，会自动补噪点、糊化、降低码率来保证流畅。
- 首帧光线差（暗光、逆光、昏暗走廊），AI 为补细节强制拉高噪点。

**解决方案：**
- 引导用户调整首帧参考图。
- **放在提示词最开头增加**：`高码率高清画质，低噪点无颗粒雪花，画面干净通透，无脏斑色块；人物皮肤哑光无油光无反光，面部色彩均匀；暗部干净无杂色，动态流畅无拖影`。
- **参考图调整**：用明亮均匀柔光，避开昏暗、逆光、侧光、暗光环境。
- 解决方案 1：将尾帧截图换成白模再用来生成视频。参考：<https://www.volcengine.com/docs/82379/2222480?lang=zh#4c6549da>
- 解决方案 2：从视频剧情角度设置分镜头，**避免使用生视频模型生成的截图用作参考图**，直接用高清的人物参考图和文字生成视频内容。


### 10.2 使用视频延长时出现多个重复人 / 双胞胎
**问题原因：** 输入参考内容过多，导致模型理解产生歧义。

**优化方案：** 直接用文字描述要延长/续写的内容。**不上传参考图，只上传要参考的视频**。多人物场景时按照唯一特征描述强调人物主体。
- 提示词参考：`向后延长视频1。要求保持人物造型不变，场景不变。生成内容：镜头一：黑衣服的男人走向绿衣服的男人说：吃了吗。镜头二：绿衣服的男人指着桌子上的饭说：我这都吃了一半了，不看不出来？`


### 10.3 视频延长画质劣化
**场景：** 将模型生成的视频作为原始视频进行延长续写时画质会劣化；多次续写会叠加，尤其在人脸会出现斑驳的色块。

**优化方案：**
1. 将原始视频用 Seedance 2.0 转化成白模视频后，再作为输入视频。
2. 【重要】使用高清的人物图和场景图作为参考。
3. 模型预计在下一版本优化。

## 12. 视频参数 / 比例问题

### 12.1 Seedance 生成的 9:16 视频放入剪映后画面对不上 / 视频黑边
**问题原因：** 放到剪映里设置 16:9 / 9:16，因为画布是标准的尺寸，视频尺寸不标准，所以画布和视频之间就有黑边。

**答复建议：**
> doubao-seedance-2-0 模型在标注 9:16 时，实际出片尺寸为了适配模型的生成对齐要求（宽高需按 16/32 像素对齐），并不是严格的 1280×720 / 1920×1080，而是经过对齐后的近似 9:16：
>
> - 480p 9:16 → 864 × 480（比例约 1.800，略宽于标准 9:16）
> - 720p 9:16 → 1248 × 704（比例约 1.773，略窄于标准 9:16）
> - 1080p 9:16 → 1920 × 1088（比例约 1.765，略窄于标准 9:16）
>
> 剪映 9:16 工程是严格的 1280×720 或 1920×1080，因此当您把上述视频拖入剪映 9:16 工程时，剪映会按工程比例自动补黑边或缩放裁切，造成"画面对不上"的现象。视频本身是无黑边的完整画面，黑边来自剪辑工程比例与素材比例的差异，并非模型输出问题。
>
> 解决方法：
> 1. 在剪映中按视频实际尺寸新建自定义工程（如 1920×1088、1248×704），可完全无黑边。
> 2. 若需严格 1080×1920 输出，建议使用 1080p 档位生成，再在剪映中裁掉上下各 4 像素到 1920×1080，画面损失最小。
> 3. 480p / 720p 与标准 9:16 的偏差比 1080p 更明显，对画框严格的场景优先选用 1080p。
