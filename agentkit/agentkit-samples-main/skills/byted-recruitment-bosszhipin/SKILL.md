# BOSS直聘招聘平台技能

## 功能概述

本技能用于在 BOSS直聘企业版平台上进行候选人推荐查看、筛选和打招呼操作。

## 前置条件：确认JD信息

在开始搜索候选人之前，**必须先使用 byted-recruitment-jd-optimization 技能确认完整的JD信息**！

### 第一步：使用JD优化技能

1. **读取 byted-recruitment-jd-optimization 技能文档**：`read ~/.openclaw/workspace/skills/byted-recruitment-jd-optimization/SKILL.md`
2. **确认JD信息**：
   - 如果已有 `/root/.openclaw/workspace/jd-process/[职位名]/[职位名]-jd.md` 文件，直接读取
   - 如果没有，按照 byted-recruitment-jd-optimization 技能的流程，与用户确认完整的JD信息
3. **获取关键信息**：
   - **显性内容**：职位名称、职责、任职要求、薪资范围、工作地点等
   - **隐性规则**：评分权重、Must项、薪资范围细节、经验要求等

### 第二步：使用隐性规则指导简历匹配

在简历匹配评分时，**必须引用JD隐性规则**：

| 评分维度 | JD隐性规则要求 | 如何应用 |
|----------|----------------|----------|
| **工作经验** | 权重：XX%，必须X年以上相关经验 | 检查候选人工作经验年限和相关性 |
| **技能匹配** | 权重：XX%，必须掌握[技能1, 技能2] | 检查候选人技能列表 |
| **学历背景** | 权重：XX%，大专/本科/硕士以上 | 检查候选人学历 |
| **行业背景** | 权重：XX%，[行业]经验优先 | 检查候选人行业经历 |
| **薪资期望** | 权重：XX%，期望范围[X]-[Y]K | 检查薪资期望是否在范围内 |
| **Must项** | 一票否决条件 | 不满足直接淘汰 |

**评分时必须同时参考：**
1. 本技能的5星评分标准
2. JD隐性规则中的权重和要求
3. Must项一票否决

## 使用流程

### 1. 登录 BOSS直聘企业版

1. **先读取 computer-use 技能文档**：`read ~/.openclaw/workspace/skills/computer-use/SKILL.md`
2. 访问 https://www.zhipin.com/web/user/?intent=1
3. 使用CUA工具打开网站：`https://www.zhipin.com/web/user/?intent=1`
4. 等待页面加载完成（建议等待2-3秒）
5. 获取页面快照查看状态
6. **如果需要登录**：
   - 检查页面是否显示登录界面
   - 如果是登录界面，移交控制权给用户手动登录
   - 告诉用户：打开登录页面 → 选择扫码登录或其他方式 → 完成登录后告诉我
7. **登录成功后**：
   - 获取快照确认登录状态
   - 可以选择保存cookies（如果浏览器工具支持）
8. **如果登录状态失效**：
   - 看到"当前登录状态已失效"提示时
   - 主动找用户重新登录

### 2. 进入【推荐牛人】/【搜索】选项卡

1. 登录成功后，查找左侧菜单的"推荐牛人"选项
2. 如果没有看到，先获取页面快照确认当前页面结构
3. 点击"推荐牛人"进入
4. 确认"职位信息"中是否有符合当前[职位名]的职位，如不完全匹配，需要进行沟通确认：
   - 职位名称（例如：BD专员）
   - 工作地点
   - 薪资范围
5. 如果需要切换职位，找到职位选择器进行切换
6. 如果"推荐牛人"中没有当前岗位，则使用"搜索"卡片搜索候选人简历

### 3. 查看候选人列表

1. 获取推荐页面完整快照
2. 逐个记录候选人基本信息：
   - **姓名**：候选人的名字
   - **年龄**：显示的年龄信息
   - **工作经验**：几年工作经验
   - **学历**：本科/大专/硕士等
   - **当前状态**：在职/离职/寻找机会等
   - **期望职位**：候选人期望的职位
   - **期望地点**：期望工作城市
   - **薪资期望**：期望薪资范围
   - **工作经历**：简要记录过往工作经历
   - **教育背景**：毕业院校和专业
   - **优势描述**：候选人的自我介绍或亮点

### 4. 候选人匹配度评分

#### 4.1 首先读取JD隐性规则

在评分前，**必须先读取对应的JD文件**：`read /root/.openclaw/workspace/jd-process/[职位名]/[职位名]-jd.md`

从JD文件中获取：
- **评分权重**：各维度的权重设置
- **Must项**：一票否决条件
- **薪资范围**：期望范围和可接受上浮
- **经验要求**：相关行业、转行条件等

#### 4.2 检查Must项（一票否决）

在详细评分前，先检查Must项：
- [ ] 学历低于XX？
- [ ] 工作经验少于XX年？
- [ ] 不掌握必备技能？
- [ ] 薪资期望超出范围？
- [ ] 地点不符合？
- [ ] 其他Must项？

**如果满足任何一项Must项，直接淘汰，不进行后续评分。**

#### 4.3 基于JD权重进行评分

根据以下标准进行评分，**同时参考JD隐性规则的权重设置**：

| 评分 | 说明 | 匹配标准（结合JD权重） |
|------|------|-------------------|
| ⭐⭐⭐⭐⭐ (5分) | 完美匹配 | 经验和技能完全符合JD要求，满足所有高权重条件，有相关行业经验，学历匹配，地点匹配，薪资期望在范围内 |
| ⭐⭐⭐⭐ (4分) | 很好匹配 | 大部分高权重条件符合，经验基本匹配，有相关技能，学历/地点/薪资基本符合 |
| ⭐⭐⭐ (3分) | 基本匹配 | 部分条件符合，有一定相关性，但需要进一步考察 |
| ⭐⭐ (2分) | 一般匹配 | 少部分条件符合，相关性较弱 |
| ⭐ (1分) | 不太匹配 | 基本不符合要求 |

**评分时必须考虑JD权重：**
- 高权重维度（如工作经验、技能匹配）占比较大
- 中权重维度（如学历、行业）占比中等
- 低权重维度（如其他加分）占比较小

**评分示例（结合JD权重）**：
- 5分：有5年外贸销售经验（权重30%），会中英阿三语（技能权重25%），正好匹配海外BD岗位
- 4分：3年电商运营经验（权重30%），熟悉抖音平台（技能权重25%），擅长B端客户开发
- 3分：2年销售经验，但行业不直接相关
- 2分：应届生，有校园活动经验，但缺乏直接工作经验
- 1分：完全不相关的工作经历

### 5. 滚动获取更多候选人

1. **执行滚动操作**：`browser action=act kind=evaluate fn="window.scrollBy(0, 1000)"`
2. **等待加载**：等待2-3秒让新内容加载
3. **获取新快照**：查看新加载的候选人
4. **继续记录**：记录新出现的候选人信息
5. **重复操作**：
   - 继续滚动 `window.scrollBy(0, 1000)`
   - 直到获取足够数量的候选人
   - 或者直到页面不再加载新内容
6. **重要提示**：
   - 列表页面有数量加载限制
   - 必须通过滚动下拉才能获取更多候选人
   - 不要只看第一页的几个候选人

### 7. 发送【打招呼】操作

对于评分最高的候选人：
1. **找到候选人卡片**
2. **查找"打招呼"按钮**（通常在候选人卡片上）
3. **点击发送打招呼**
4. **记录已发送打招呼的候选人**
5. **建议操作**：
   - 优先给评分最高的10位候选人发送
   - 先从5星候选人开始，然后是4星
   - 记录发送名单

## 注意事项

### 登录相关

1. **登录安全**：
   - 不要自动化输入登录凭据
   - 使用 `<browser-handoff />` 移交控制权
   - 告诉用户手动完成登录

2. **Cookies管理**：
   - 登录后可以尝试保存cookies
   - 方便后续使用避免重复登录

3. **登录失效处理**：
   - 如果显示"当前登录状态已失效"
   - 立即找用户重新登录
   - 不要尝试绕过登录
   - 不要自动重复尝试登录

4. **安全验证**：
   - 如果遇到滑块验证、验证码等
   - 移交控制权给用户完成
   - 不要尝试自动化处理

### 页面加载和跳转问题

1. **页面重定向**：
   - 如果页面跳转到百度首页或其他页面
   - 不要惊慌，这是常见问题
   - 重新打开BOSS直聘URL：`https://www.zhipin.com/web/user/?intent=1`
   - 如果问题持续，移交控制权给用户手动操作

2. **网络问题**：
   - 如果页面加载失败
   - 等待2-3秒后重试
   - 检查网络连接状态
   - 重试3次后停止重试

3. **页面加载等待**：
   - 每次操作后等待2-3秒
   - 让页面完全加载
   - 然后再获取快照或继续操作

### 候选人筛选

1. **职位选择**：
   - 在【推荐牛人】选项卡中
   - 逐个选择需要招聘的【职位】
   - 确认职位信息正确
   - 如果【推荐牛人】选项卡中没有对应【职位】，则使用【搜索】功能搜索候选人简历

2. **数量要求**：
   - **每个职位至少找10个评分较高的候选人**
   - 高匹配度指4星和5星候选人
   - 如果不够，继续滚动获取更多

3. **信息完整度**：
   - 尽量记录完整的候选人信息
   - 包括工作经历、教育背景、优势描述等
   - 这些信息对评分很重要

4. **滚动加载技巧**：
   - 每次滚动 `window.scrollBy(0, 1000)`
   - 不要一次性滚动太多
   - 给页面足够时间加载新内容

### 打招呼操作

1. **优先级策略**：
   - 优先给评分最高的候选人发送
   - 5星 → 4星 → 3星的顺序
   - 确保高匹配度候选人优先被联系

2. **数量控制**：
   - 建议给评分最高的10位候选人发送
   - 可以根据实际情况调整
   - 不要一次性发送太多避免被限制

3. **记录跟踪**：
   - 详细记录已发送打招呼的候选人
   - 包括姓名、评分、发送时间
   - 便于后续跟进

### 常见问题处理

| 问题 | 解决方案 |
|------|----------|
| 页面跳转到百度 | 重新打开BOSS直聘URL，或移交控制权给用户 |
| 登录状态失效 | 找用户重新登录 |
| 安全验证 | 移交控制权给用户手动完成 |
| 页面加载慢 | 等待2-3秒，或刷新重试 |
| 候选人数量不够 | 继续滚动页面加载更多 |
| 找不到"推荐牛人" | 获取快照查看页面结构，确认位置 |

## 实际操作经验总结

### 成功经验
1. **先读云桌面技能**：每次都先读 `computer-use/SKILL.md`，这是硬性要求
2. **充分等待**：页面加载后等待2-3秒再操作，避免超时
3. **快照确认**：每次关键操作后都获取快照，确认状态
4. **滚动技巧**：每次滚动1000像素，不要贪多
5. **详细记录**：候选人信息越详细，评分越准确
6. **文件保存**：用Markdown格式保存，结构清晰易读

### 避坑指南
1. **不要跳过云桌面技能**：必须先读 `computer-use/SKILL.md`
2. **不要自动化登录**：遇到登录就移交控制权
3. **不要只看第一页**：必须滚动获取更多候选人
4. **不要忽略评分标准**：严格按5星标准评分
5. **不要忘记保存文件**：完成后立即保存候选人信息
6. **不要跳过重定向问题**：页面跳转时重新打开或移交控制权
