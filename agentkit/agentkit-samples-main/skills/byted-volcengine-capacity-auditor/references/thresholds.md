# 巡检阈值口径

## 通用风险分级

- `< 70%`：无明显容量风险
- `70% - 80%`：低风险，建议关注
- `80% - 90%`：中风险，建议预警
- `>= 90%`：高风险，建议优先处理

## ECS

### 低利用率候选

同时满足以下条件时，可进入缩容候选池：

- CPU `p95 < 20%`
- 内存 `p95 < 30%`
- 磁盘使用率 `p95 < 50%`

### 风险关注

- CPU `p95 >= 70%`
- 内存 `p95 >= 70%`
- 磁盘使用率 `p95 >= 70%`
- 如果已知 IOPS 上限，`IOPS >= 85%` 视为风险信号

## RDS MySQL

### 当前 V1 可用口径

- 当前快照：`NodeCPUUsedPercentage / NodeMemoryUsedPercentage / NodeSpaceUsedPercentage`
- 近 30 天：`QPS / ConnUsage`

### 低利用率候选

满足以下任意组合时，可列入观察名单：

- 当前 CPU `< 20%` 且当前内存 `< 30%`
- 当前磁盘 `< 50%`
- 近 30 天 `ConnUsage` 长期接近 `0`
- 近 30 天 `QPS` 平稳且峰值显著低于实例预期承载

### 风险关注

如果后续补齐历史 CPU / 内存 / 磁盘时序，则沿用通用风险分级。

## CLB

### 当前 V1 可用口径

- 仅资源清单

### 后续建议补齐

- 带宽使用率
- 并发连接使用率
- 新建连接使用率
- 七层 `QPS` 使用率
- 平均响应时间

在这些指标未补齐之前，CLB 不输出结论性扩缩容建议。
